Audit the state management of this codebase and fix all anti-patterns.

FIND AND FIX PROP DRILLING:
  - Props passed through 3+ component levels without being used in between
  - Fix: lift the state to context or a store, consume directly where needed

FIND AND FIX DUPLICATED STATE:
  - The same piece of data stored in useState in two different components
  - Fix: single source of truth — lift to a shared context or store

FIND MISSING GLOBAL STATE:
  - Auth/user info stored in local useState (should be in AuthContext or store)
  - Theme/locale stored in a component (should be global)
  - Cart, session, notifications — anything cross-cutting should be in store/context

FIND OVER-USED GLOBAL STATE:
  - Things in a global store that are only ever used in one component
  - Fix: move back to local useState

STATE PLACEMENT RULES TO ENFORCE:
  - Local UI state (isOpen, activeTab, inputValue) → useState in component
  - Shared state used by 2-4 components → Context
  - Complex shared state, async actions, many consumers → Zustand/Redux slice
  - Server state (data from API) → custom hook with loading/error/data

OUTPUT:
  List every violation with the fix.
  Then apply the fix — new context file, updated components, removed prop drilling.
